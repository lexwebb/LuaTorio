## Tooling

```bash
pnpm rip:cookbook
# → research/cookbook-2.0/{catalog.json,patterns.md,out/,raw/}
```

Script: `scripts/rip-cookbook-2.0.mjs` — paginates the forum thread, extracts every `div.bptext` blueprint string, decodes with `@jensforstmann/factorio-blueprint-tools`, tags features (EACH, R/G network split, dual-color wiring, copy+const, else_outputs, feedback, selector ops).

Bulk `out/` + `raw/` are gitignored; `catalog.json` + `patterns.md` are committed.

**Surprise encoding detail:** cookbook clocks often omit `copy_count_from_input` for the copy channel and set it `false` (no `constant` field) for the +1 channel. Analyzer treats that as copy+const.

**else_outputs** are rare in this thread (many holds use silence + feedback instead). Our #50 else-hold path is still valid; cookbook prefers “then only” clocks that wrap.
